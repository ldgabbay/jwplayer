Advertising In JW Player

This document describes the various advertising options available in the Ads edition of JW Player.

Ad Types

Out of the box, JW Player supports two ways to deliver in-video ads:

Direct Campaigns
Video pre-rolls, text overlays and image end-caps, configured by setting creative URLs and click targets. An ad-server is not needed. These options are great for one-off campaigns, house ads or call-to-actions for marketing or e-commerce. See Running Direct Campaigns for more info.
Google IMA
in-video ads served by Google's Dart For Publishers (DFP) advertising server, including TruView prerolls and AdSense text overlays. JW Player is one of Google's official video technology partners. See Running Google IMA Ads for more info.

Additionally, several advertising networks have integrated their client-side technology with JW Player. Examples are Tremor, AdoTube and Adap.tv. Documentation for these integrations can be found on the websites of these partners.

VAST/VPAID Support

If you need VAST/VPAID support in JW Player, Please contact us for more information.

Configuration

Advertising in JW Player is so far only supported in Flash mode. Therefore, always set the primary configuration option to flash. Next, configure ads using the advertising block:

jwplayer('myElement').setup({
    primary: 'flash',
    file: 'assets/myVideo.mp4',
    advertising: {
        // Advertising options
     }
});

See Embedding JW Player for general info on player embeds. See the documentation of the various ad types for info on the configuration options they provide.

Note it is not possible to combine multiple ad types (e.g. VAST/VPAID and Tremor, or Direct and IMA) in a single embed. Multiple ads per embed are supported, of course.
