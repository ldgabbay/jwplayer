Using JW Player Skins

This guide explains how to get and configure existing skins for JW Player. If you're looking for information on how to design and package a skin yourself, see Building JW Player Skins.

Introduction

JW Player skins change the appearance of the player, making it possible to match your embed to your site's aesthetic. All visual components of the player (like the controlbar, display and playlist) can be skinned. Here are a few examples of a skinned player:

Technically, a JW Player skin consists of an XML file that contains a number of settings (e.g. the color of the tooltip texts) and a number of links to PNG images (e.g. for the play button). The XML + PNG files are packed into a single file to speed up loading:

  • For Flash, the XML file and PNG images are compressed into a single ZIP file.
  • For HTML5, the PNG images are embedded into the XML file as base64 encoded strings.

Getting a Skin

There are two ways to get a skin for your player:

  • If you are proficient with an image editor (like Photoshop), designing and packaging your own JW Player skin is straightforward. See Building JW Player Skins for more info.
  • The higher editions of JW6 include a pack of 8 skins designed by us. You are free to use these skins on your websites, or to modify these skins (by unpacking the ZIP files and editing the images).

After downloading (or packaging) the skin of your choice, you are left with its XML and ZIP files (e.g. bekle.xml and bekle.zip). Upload these two files to your webserver, so the skin can be configured for your player.

Configuring a Skin

Once you've uploaded your skin, simply set the skin configuration option to the path where you uploaded the ZIP file. This can be:

  • An absolute path (e.g. http://example.com/skins/bekle.zip)
  • A domain-absolute path ((e.g. /skins/bekle.zip)
  • A path relative to the HTML file (e.g. ../skins/bekle.zip)

In HTML5 mode, the player automatically presumes an XML file with the same name as the ZIP is available. So if you link to your ZIP file as /skins/bekle.zip, the player will load /skins/bekle.xml in HTML5 mode.

Example setup

Here is an example player setup block, using a domain-absolute path to the skin files:

jwplayer("myElement").setup({
    file: "/uploads/example.mp4",
    height: 360,
    image: "/uploads/example.jpg",
    skin: "/uploads/bekle.zip",
    width: 640
});

Note that both the XML and ZIP file of a skin are subject to Crossdomain Security Restrictions, which causes issues when loading the skins from another domain than your website's.
