Quick Start

Getting JW Player up and running is now easier than ever. Only a few lines of HTML code is needed to embed a video that works on any desktop browser and mobile device. This guide walks through the necessary steps.

Uploading the Files

In order to embed a video, you first need the file uploaded to your server. Ensure it is in the MP4 format, since that can be played on any browser and device (including iPad, iPhone and Android).

  • If your video is not in the MP4 format, you can convert it using a free tool like Handbrake. See our Video Encoding guide for more info.
  • If you have no video at hand, feel free to use this short clip for your testing.

Second, you should grab JW Player from our site and upload it to your webserver. The following three files need uploaded and placed in a directory that's accessible from the internet:

  1. jwplayer.js: the overall embed library.
  2. jwplayer.html5.js: the actual HTML5 player.
  3. jwplayer.flash.js: the actual Flash player.

Third, you should upload a poster image to your server. This image (in JPG format) is displayed before the video starts. An interesting frame from the video usually works well to draw attention and entices users to click play. Here is an image for our short test clip.

Embedding the Player

With all files uploaded to your site, embedding the JW Player is a simple, 3-step process. Open your web page in an HTML editor and:

  1. Load the JW Player embed library from the head of your page: 
    <script type="text/javascript" src="/scripts/jwplayer.js" ></script>
  2. Insert a <div> element with a unique id at the location you want the player to appear: 
    <div id="myElement">Loading the player ...</div>
  3. Insert a script that invokes the JW Player setup() call on your <div>:
    4. <script type="text/javascript">
    jwplayer("myElement").setup({
        file: "/uploads/example.mp4",
        height: 270,
        image: "/uploads/example.jpg",
        width: 480
    });
</script>

In the setup() block, file and image should be the URLs to your video file and poster images. Their paths should be relative to the HTML page (just like when embedding an image using <img>). The height and width are used to set the dimensions of the player. Feel free to change these to values that make the player nicely fit your page.

  • Working with Playlists explains how to group multiple videos into a playlist.

  • Using the JavaScript API explains how to build rich player interactions with JavaScript.
